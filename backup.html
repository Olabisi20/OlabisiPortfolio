Workflow

List of projects and roles I worked on as a Data Storage and Protection Engineer:

  1. On Call Cordinator
  2. Add Client to Backup using CLI and GUI
  3. Backup Failure Troubleshooting
  4. Restore Job from Client
  5. Decomission Client(s)
  1. On Call Cordinator: Summmary of duties I performed while on call
    • Monitor servers, alerts and reports (NAS/SAN/BUR/STORAGE)
    • Enagage in P1/P2 incident tickets
    • Triage process
    • Engage Vendor
    • Represent Team on Technical issues
    • Escalate to product specialist if required
  2. Add Client to Backup using Command Line and GUI: After the problem has been understood, define the purpose of the story/ problem, then create test cases to catch all the necessary use cases
    • Add Client to Backup using command Line: Login to server then run this commands - 'nslookup', 'clientname', 'ifconfig', 'nsr clientname'.
    • Add Client to Backup using GUI (Networker): Login to Networker, click on add, then add the client name and aliases, fill the protection group list to add to daily and time you want the backup to run and save.
  3. Backup Failure Troubleshooting: Sometimes we backup some client and the daily reports shows they have not been backing up possible reasons and fix are:
    • Start networker service: Linux: #service networker start” or #systemctl status networker HP UX: #/sbin/init.d/networker start
    • Install networker client: If backup is not running we need to check if networker client is installed Linux: #rpm -qa | grep lgto* HP UX: #swlist -l subproduct | grep -i networker
    • Networker Client failing to start: Could be because there is no free ports, Use “netstat -an wc -l” to get a count of the open ports
    • No route to host: Add route to host
    • Filesystem backup exceeds 90% usage, backup should not exceed 90% usage in order for backup to run successfully
  4. Restore Job from Client: command prompt to restore job from client - $ cp -R source_folder destination_folder cd to the folder below that folder ch add
    • -s: Server
    • -iN: if file already exist, skip recovering file
    • -c: Client
    • -iY: Overwrite existing if it already exists
    • -R: Destination Client
    • -iR: Rename recovered file if name exists (better to use than overwrite iY)
  5. Decommission Client: When a client is no longer needed or retired, we need to get them decommissioned so we do not coontinually run backups on the client - Login to Networker, Search for client name to decommissioned, then under the protection group list select Decommission and Save.
